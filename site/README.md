# Starboard – Site

The Starboard dashboard is a React + TypeScript SPA that visualises and manages your GitHub starred repositories.

## Tech Stack

- **React 19** with TypeScript (strict mode)
- **Vite** as build tool (base path `/starboard/` in production)
- **TailwindCSS v4** via `@tailwindcss/vite`
- **Zustand** for global state management
- **@tanstack/react-virtual** for virtualised table rendering
- **react-router-dom** for client-side routing and URL sync

## Data Source

The dashboard reads `data/stars.json` at runtime from the Vite public directory (`site/public/data/`). This file is generated by the Python pipeline (`scripts/fetch_stars.py` + `scripts/enrich_stars.py`) and copied into place by `scripts/build_site.py`.

## Local Development

```bash
cd site
npm install
npm run dev
```

The dev server runs at `http://localhost:5173/` (base `/` in dev mode).

## Production Build

```bash
cd site
npm run build
npm run preview
```

The production build outputs to `site/dist/` with base path `/starboard/` for GitHub Pages deployment.

## Deployment

Deployment is handled automatically by the GitHub Actions workflow (`.github/workflows/refresh.yml`), which:

1. Fetches starred repos via `fetch_stars.py`
2. Enriches with LLM metadata via `enrich_stars.py`
3. Copies `stars.json` to `site/public/data/` via `build_site.py`
4. Builds the site with `npm run build`
5. Deploys `site/dist/` to GitHub Pages
